# SUB_SKILL: Script Analysis

将广告/电商脚本拆解为结构化的单元（Unit）和子分镜（Sub-shot），输出 JSON 结构，供后续生图/视频工具使用。

## 输入

```json
{
  "raw_script_text": "客户已通过的脚本文本（含镜号、旁白、画面描述、时长等）",
  "tts_durations": [24, 19, 9],
  "platform": "douyin"
}
```

只有 `raw_script_text` 必填，其余自动推断。

| 字段 | 默认值 | 说明 |
|---|---|---|
| `tts_durations` | `[]` | 各镜号 TTS 精确时长（秒），未提供则按旁白字数估算（中文 4 字/秒） |
| `platform` | `douyin` | 目标平台，影响分镜节奏 |

## 内容来源优先级

| 优先级 | 来源 | 权重 | 作用 |
|---|---|---|---|
| **第一优先** | **旁白**（实际音频内容） | **主** | 拆解旁白含义 → 决定每个子分镜展示什么画面 |
| **第二优先** | 脚本中的画面描述 | **辅** | 辅助理解用什么视觉元素表达旁白含义 |

**画面服务于旁白的含义。** 不是机械复刻脚本的画面描述，而是根据旁白在说什么，推导观众此刻应该看到什么。

关键规则：
- 旁白讲"问题/痛点" → 画面只展示问题，不提前出现品牌产品
- 旁白讲"竞品/传统方法缺点" → 画面只展示竞品，品牌不登场
- 旁白讲"我们的解决方案" → 品牌产品才正式出现

## 处理流程

### Step 1：脚本判题

从脚本中提取 4 个维度：
- `topic`：核心主题（如"氨基酸洗发水去屑"）
- `product_category`：品类（如"洗护"）
- `style_anchor`：风格锚点（写实 / 科普 / 电竞 / ...）
- `key_actions`：关键动作清单（可拍摄的动作）

### Step 2：拆单元 + 判型

按镜号边界拆成 U01/U02/...，并判型：

| 类型 | 含义 | 默认节奏 |
|---|---|---|
| `T1` | 数字人口播（讲解为主） | 3s/镜 |
| `T2` | 动作展示（操作切换为主） | 2s/镜 |
| `T3` | 动画科普（机制链路为主） | 2s/镜 |

低置信时回退 T2。

### Step 3：拆单元规则

**时长基准：以 TTS 语音录音的精确时长为准**，不以脚本标注时长为准。如果尚未生成 TTS，按旁白字数估算：中文约 4 字/秒。

#### 长脚本（>45s）

每个镜号内容独立性强，直接**每个镜号 = 一个单元**：

```
镜号 1（TTS 24s）→ 单元 1 → 段 A(15s) + 段 B(9s)
镜号 2（TTS 19s）→ 单元 2 → 段 A(15s) + 段 B(4s)
镜号 3（TTS  9s）→ 单元 3 → 完整段
```

#### 常规脚本（20-45s）

多个镜号可归组到同一个 15s 单元内：
1. 按叙事段落优先分组（开场/痛点/产品/效果/收束）
2. TTS 累计时长不超过 15s 为硬约束
3. 叙事段落边界和 15s 边界冲突时，以叙事完整性优先，允许 ±2s 浮动

```
镜号 1+2+3（TTS 共 14s）→ 单元 1（开场+痛点）
镜号 4+5  （TTS 共 12s）→ 单元 2（产品+效果）
镜号 6    （TTS     8s）→ 单元 3（收束）
```

#### 短单元处理

即使单元只有 9s 甚至更短，也**独立成单元，不与相邻单元合并**。每个单元对应一个独立的视频生成任务，混合不同叙事内容会降低生成质量。

### Step 4：超 15s 单元的段拆分

当一个单元 TTS 时长超过 15s：
1. 在旁白自然停顿处（句号/逗号）切分为段 A（<=15s）+ 段 B（剩余）
2. 段 A 和段 B 各自独立出资产包
3. 段 B 标注 `continuation_from` 字段
4. 两段归属同一个单元编号

### Step 5：分镜节奏

每个单元内部按以下节奏拆子分镜：

| 分镜节奏 | 每镜时长 | 15s 可容纳 | 适用内容 |
|---|---|---|---|
| 快切 | 1-1.5s | 10-15 镜 | 产品快闪、纯切换 |
| **标准（默认）** | **1.5-2s** | **8-10 镜** | **大多数电商分镜** |
| 慢动作 | 2.5-3s | 5-6 镜 | 倒水、涂抹、泡沫扩散等复杂动作 |

分镜数量 3 维度决策：

| 维度 | 说明 | 优先级 |
|---|---|---|
| ① 旁白内容含义 | 旁白在说什么 → 观众需要看到几个画面 | 第一优先 |
| ② 脚本推荐的画面/动作 | 脚本描述了多少个可视化场景 | 参考 |
| ③ 时间节奏 | 按 1.5-2s/镜估算，复杂动作 3s/镜 | 兜底 |

**原则：镜头数尽可能做多。** 快节奏 = 画面切换快 = 信息密度高 = 更抓眼球。

## 人物出镜规则

### 创始人 / 核心角色

| 规则 | 说明 |
|---|---|
| **仅在旁白明确要求出镜时才安排** | 如"大家好我是XX"等自我介绍 |
| **集中安排，不碎片化** | 集中在 1-2 个单元的完整段中 |
| **旁白 ≠ 出镜** | 声音全程覆盖，画面只在需要时出现 |
| **其余单元用 B-roll** | 填充消费者、CG、产品、场景等 |

### 素人 / 消费者

自由出现，无需跨单元一致性。不同单元可使用不同素人。

## 输出格式

### 单单元（<=15s）

```json
{
  "unit_id": "unit_01",
  "shot_ids": [1, 2, 3],
  "tts_duration": 14,
  "unit_type": "T2",
  "narrative_role": "开场+痛点",
  "sub_shots": [
    { "panel": "P1", "time_range": "0-2s", "subject": "办公桌", "action": "键盘文件杂乱", "purpose": "Hook" },
    { "panel": "P2", "time_range": "2-4s", "subject": "女生", "action": "伸手够杯子被打断", "purpose": "痛点" }
  ],
  "grid_size": "3x3",
  "has_human_face": true
}
```

### 多段单元（>15s）

```json
{
  "unit_id": "unit_01",
  "shot_ids": [1],
  "tts_duration": 24,
  "unit_type": "T1",
  "narrative_role": "创始人自述",
  "segments": [
    {
      "segment_id": "seg_a",
      "time_range": "0-15s",
      "sub_shots": [...],
      "grid_size": "3x3"
    },
    {
      "segment_id": "seg_b",
      "time_range": "15-24s",
      "sub_shots": [...],
      "grid_size": "2x3",
      "continuation_from": "seg_a"
    }
  ],
  "has_human_face": true
}
```

```json

```

## 宫格尺寸自适应

根据子分镜数量自动选择宫格尺寸：

| 子分镜数 | 推荐宫格 | 布局 | 适用场景 |
|---|---|---|---|
| 3-4 | 4 格 | 2x2 | 极短视频（5-8s） |
| 5-6 | 6 格 | 2x3 | 短视频（10s） |
| 7-9 | 9 格 | 3x3 | 标准 15s 视频 |
| 10-12 | 12 格 | 3x4 | 详细 15-20s |
| 13-16 | 16 格 | 4x4 | 长视频（20-30s） |
| 17-25 | 25 格 | 5x5 | 30s+ 完整广告 |

**宁少不多。** 5 个子分镜用 6 格比用 9 格好 — 空格会导致生图工具填充无关内容。
