# SUB_SKILL: Face Bypass

视频生成平台人脸检测绕过方案全集。包含 5 种已验证方案和决策树，覆盖从官方通道到电商实战的全部场景。

## 下游对接

可自动化的方案（B 风格转绘、E 虚拟演员）会输出本技能兼容的 compact JSON，直接传给 `scripts/image/submit.py` 执行。手动方案（A 形象克隆、C 网格覆盖、D ID 引用）标注为 [手动步骤]。

## 背景

主流视频生成平台（如 Seedance 2.0）自 2026 年 2 月起**禁止直接上传真人脸照片作为参考图**。上传写实人脸会被系统拦截。

原因：中国《深度合成管理规定》要求 AI 编辑他人肖像必须获得同意，平台采取保守策略在上传环节直接拦截。

**但有 5 种方法可以实现真人出镜。**

## 快速决策树

```
需要真人面容吗？
├── 不需要 → 方案 E（AI 虚拟演员）
└── 需要
    ├── 本人可以在手机上录制？ → 方案 A（形象克隆，最佳）
    └── 本人无法录制，只有真人照片
        ├──【默认】电商出片 → 渔网袜 prompt 法（宫格生图自动处理，已验证 ✅）
        ├── 正式商用/品牌广告？ → 方案 C-2（局部重绘，输出最干净）
        ├── 要求极度像本人？ → 方案 B（风格转绘 + 写实反转）
        └── 需要极高保真 + 在即梦网页端？ → 方案 D（ID 引用）
```

## 方案 A：平台官方形象克隆 [手动步骤]

平台**官方提供**的真人出镜方案。通过录音录像完成本人身份校验后，生成经过认证的"数字人分身"。

```
操作步骤（手动，在即梦 APP 中完成）：
1. 打开即梦 APP（必须是手机 APP，网页端不支持）
2. 点击右下角"我的" → "创建分身" 或 "管理分身"
3. 点击"开始录制"，正脸对准前置摄像头（可开美颜）
4. 按提示朗读 "123 456 789"，同时缓慢左右转头、轻微点头
5. 等待"录入成功"，分身创建完成
6. 生成视频时选中分身，或在提示词中 @分身名
7. 平台以真实面容为基础生成视频
```

关键细节：
- **仅限本人**：系统绑定账号
- **可授权他人**：可以授权朋友使用你的分身创作
- **平台限制**：仅即梦 APP 和豆包 APP 支持
- **合规性**：完全合规，官方通道
- **一致性**：最高

## 方案 B：风格转绘 + 写实提示词反转

先把真人照片 AI 转绘为插画/CG 风格（绕过检测），再在视频提示词中要求"写实摄影"风格（反转回接近真人的效果）。

操作步骤：
1. 准备真人高清正面照
2. 通过本技能 `scripts/image/*` 做图生图风格转绘
3. video_prompt 中加入"写实摄影风格、实景拍摄、真人质感"

image-generation JSON：
```json
{"prompt":"保持人物五官特征和发型，转为高清写实商业插画风格，半身照，纯白背景","model":"nano2","aspect_ratio":"3:4","resolution":"1K","reference_images":["<真人照片路径>"]}
```

控制相似度参数（建议 0.6-0.8），得到"保留五官但不被识别为真人"的风格化图片，然后用作视频生成的参考图。

风格化强度参考：
- 0.5-0.6：明显 AI 风格，五官轮廓像本人
- **0.6-0.8**：接近真实但带轻微 AI 感（**推荐**）
- 0.8+：高度还原，可能被拦截

## 方案 C：图片预处理过检测

### C-1：渔网袜 prompt 法（电商出片首选，已验证 ✅）

在宫格生图 prompt 中直接要求生图模型在人脸区域覆盖黑色细款渔网袜图案。生图模型不拦人脸，宫格图正常生成。上传视频生成平台时，渔网袜阻断人脸检测器的特征点识别。视频生成时模型自动忽略网格，输出干净人脸。

prompt 写法：
```
凡出现人物面部的分镜，保持人物表情发型，面部覆盖黑色细款渔网袜，网眼均匀，不遮挡轮廓，
身体、服装、头发和产品不做任何覆盖处理
```

video_prompt 追加：
```
参考图中人物面部的网状覆盖物仅为参考标记，视频中人物面部必须干净清晰，皮肤光滑自然，无任何网格、网纱、遮挡物
```

写法要点：
- **必须写"黑色细款渔网袜"**，生图模型对此描述识别最稳定
- **必须写"网眼均匀，不遮挡轮廓"**
- 身体/服装/头发/产品不加覆盖，只处理面部

哪些画面需要加：

| 画面内容 | 被拦概率 | 是否加 |
|---|---|---|
| 正面大头照、眼神直视 | 高 | 必须加 |
| 半身照、自然表情 | 中高 | 必须加 |
| 3/4 侧脸 | 中 | 加 |
| 侧脸 profile | 低 | 可不加 |
| 远景全身（脸部像素小） | 很低 | 不需要 |
| 背影 / 手部 / 产品特写 | 不会拦 | 不需要 |

### C-2：手动网格覆盖法 [手动步骤]

在参考图上叠加白色网格线，破坏五官空间关系。

```
操作步骤（手动，在图片编辑工具中完成）：
1. 打开网格工具（在线工具或 Photoshop）
2. 上传真人参考图
3. 设置网格参数：起步推荐 6×6 白色网格，线宽 12px，不透明度 100%
4. 网格线必须是 100% 不透明实线（半透明不行，检测器能看穿）
5. 下载处理后的图片，作为视频生成的参考图
6. video_prompt 中加入："画面干净无网格线，皮肤光滑自然，无覆盖物"
```

参数速查：

| 场景 | 网格 | 线宽 | 效果 |
|---|---|---|---|
| 侧脸/远景 | 4×4 | 15px | 最少伪影，可能过不了正脸 |
| **标准（推荐）** | **6×6** | **12px** | **可靠通过 + 伪影可控** |
| 高清正脸 | 10×10 | 8px | 最可靠，伪影较多 |

伪影处理：用 `fast` 模式做草稿测试，确认后换 `pro` 出正片。残留网格线可在剪映用"美颜/磨皮"消除。

### C-3：脸部局部重绘（ComfyUI / Photoshop）

只对脸部做轻度风格化，身体完全不动。

ComfyUI：
1. 用人脸检测节点生成脸部 mask，羽化 5-10px
2. 对 mask 区域做 img2img，denoising 0.3-0.5
3. 提示词："同一人物，轻度商业插画风格，保留五官特征"
4. video_prompt 写"写实摄影风格，真人质感"

Photoshop 简化版：
1. 套索工具选中脸部
2. 滤镜 → 模糊 → 高斯模糊（半径 3-5px）
3. 或 滤镜 → 风格化 → 油画效果（轻度）

denoising 强度参考：
- 0.2-0.3：几乎看不出变化，可能过不了
- **0.3-0.5**：轻微 AI 感，五官高度保留（**推荐**）
- 0.5-0.7：明显风格化，检测必过但可能不像本人

### C-4：风景叠加法

```
1. 准备人物照片 + 风景图（树林/天空/城市）
2. Photoshop 中把风景图叠在人物图上，不透明度 30-50%
3. 人脸被风景像素"污染"，检测置信度降低
4. video_prompt 描述清晰人物和场景，模型会忽略叠加风景
```

### 批量处理脚本（ImageMagick）

```bash
for img in input/*.jpg; do
  convert "$img" \
    -stroke white -strokewidth 12 \
    $(for i in $(seq 1 5); do echo "-draw \"line $((i*width/6)),0 $((i*width/6)),height\""; done) \
    $(for i in $(seq 1 5); do echo "-draw \"line 0,$((i*height/6)) width,$((i*height/6))\""; done) \
    "output/grid_$(basename $img)"
done
```

## 方案 D：平台 Agent 模式 ID 引用 [手动步骤]

在平台 Agent 模式中让 AI 先生成人物图，获取内部 ID。用 ID 引用的图片被标记为"AI 生成"，跳过人脸检测。

```
操作步骤（手动，在平台网页端完成）：
1. 平台网页端 → Agent 模式
2. 让 Agent 生成理想人物形象图
3. 将图片拖入参考图区域
4. 让 Agent 输出图片 ID（形如 #img_xxxxx）
5. 提示词引用 ID："角色与 #img_xxxxx 保持完全一致"
6. 切换到视频生成模式
```

限制：
- 仅限平台网页端，不支持 API 自动化
- 平台可能随时调整机制
- 每次新会话需重新获取 ID

## 方案 E：AI 生成虚拟演员

不需要真人面容时的最佳选择。通过本技能 `scripts/image/*` 生成虚拟人物形象。

image-generation JSON：
```json
{"prompt":"25岁亚洲女性，长直黑发，白色T恤，自然微笑，纯白背景，半身照，高清写实","model":"nano-pro","aspect_ratio":"3:4","resolution":"1K"}
```

下载生成结果作为"虚拟演员定妆照"，后续所有生图和视频生成都用这张图做 `reference_images` / `image_url` 参考，模型自动保持一致性。

## 人物出镜规则

### 创始人 / 核心角色

| 规则 | 说明 |
|---|---|
| 仅在旁白明确要求出镜时才安排 | 如自我介绍、手持产品演示 |
| 集中安排，不碎片化 | 集中在 1-2 个单元的完整段中 |
| 旁白 ≠ 出镜 | 声音全程覆盖，画面按需出现 |
| 其余单元用 B-roll | 消费者、CG、产品、场景等 |

原因：创始人形象需跨单元保持外貌一致，碎片化分布导致每个单元都需独立处理一致性，成本极高。

### 素人 / 消费者

自由出现，无需跨单元一致性。仍需人脸过检处理（视频生成平台不区分创始人和素人）。

## 电商场景推荐

| 场景 | 推荐方法 | 原因 |
|---|---|---|
| **标准电商出片** | **渔网袜 prompt 法** | 全自动，零手动，已验证 |
| 快速出片 | 渔网袜 prompt 法 | 无额外步骤 |
| prompt 效果不佳时 | 手动网格覆盖 | 兜底，10 秒搞定 |
| 品牌广告正式商用 | 局部重绘（ComfyUI） | 输出最干净 |
| 批量处理 | 手动网格（ImageMagick） | 一行命令批量处理 |

## 与长视频的组合策略

| 视频类型 | 人脸方案 | 续接方式 | 操作 |
|---|---|---|---|
| 30s 品牌广告，创始人出镜 | A（形象克隆） | 链式续接 | 分身 → 视频延伸 |
| 30s 远程代言人 | B（风格转绘） | 链式续接 | 转绘 → 参考图 → 链式 |
| 30s 电商模特 | 渔网袜 prompt | 链式续接 | 自动处理 → 链式继承 |
| 45s 连续短剧 | A 或 D | 链式续接 | 优先 A，否则 D |
| 30s 产品展示无人脸 | E 或不需要 | 共享锚定 | 并行生成 |
