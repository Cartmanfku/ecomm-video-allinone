# ONBOARDING — 三种使用方式

安装本技能后，根据你手头的材料，选择下面 **一种** 方式开始即可。三种方式最终都会汇入同一套流水线（脚本分析 → 宫格生图 → 视频描述 → 可选执行生成）。

---

## 方式 1：直接输入脚本

**适合：** 已有定稿脚本（含镜号、旁白、画面描述、时长等），想直接出宫格图和视频生成指令。

**你需要提供：**

- 脚本文本（粘贴即可）
- 可选：产品参考图、目标平台、TTS 各镜时长、宫格尺寸

**示例说法：**

> 这是我的 15 秒广告脚本，帮我拆成宫格并出视频提示词。  
> 镜号 1：……旁白：……画面：……

**流程：**

```
脚本文本
  → Step 0–5（脚本分析 → 宫格 → 视频描述 → 质检）
  → 输出 image_gen_input + video_gen_input 资产包
  → （可选）执行 scripts/image/*、scripts/video/* 真正出片
```

**详细规则：** `SKILL.md` →「有脚本模式」；`docs/sub_skills/script-analysis.md`

---

## 方式 2：输入参考视频，拆解出脚本

**适合：** 有竞品/参考短视频，想先还原或仿写脚本，再生成自己的视频资产。

**你需要提供（任选其一）：**

- 消息里附上视频文件（`media/...mp4`）
- 抖音链接（`douyin.com` / `v.douyin.com`）
- DeskClaw 已上传的 `video_id`

**示例说法：**

> 分析这个参考视频，输出原视频脚本。  
> 把这个抖音链接拆解成脚本，然后帮我生成新视频的宫格和提示词：https://v.douyin.com/xxxx/

**流程：**

```
参考视频 / 抖音链接 / video_id
  → video-analysis（上传 → 拆解 → 输出 script_text）
  → script_text 作为 raw_script_text
  → Step 0–5 → 资产包 → （可选）执行生成
```

**注意：**

- 要「原脚本」时说清楚，不要误走仿写流程
- 抖音链接需先下载并真实拆解，不能仅凭页面标题猜测脚本
- 依赖：`pip install -r requirements.txt`（`requests`、`yt-dlp`）

**详细规则：** `docs/sub_skills/video-analysis.md`

---

## 方式 3：输入产品图和卖点，自动生成脚本

**适合：** 只有产品和卖点，还没有脚本，需要从 0 写脚本并出完整资产包。

**你需要提供：**

- 产品图（至少 1 张）
- 产品名称
- 核心卖点（列表）
- 可选：目标时长（默认 15s）、目标平台

**示例说法：**

> 我有产品图，帮我做一条 15 秒抖音广告。  
> 产品：电竞桌；卖点：L 型转角、升降功能、理线槽。

**流程：**

```
产品图 + 名称 + 卖点
  → 自动生成脚本（时长 → 镜头数 → 宫格尺寸）
  → Step 0–5 → 资产包 → （可选）执行生成
```

**时长与宫格参考：**

| 目标时长 | 建议镜头数 | 建议宫格 |
|---|---|---|
| 5–8s | 3–4 镜 | 4 格（2×2） |
| 10s | 5–6 镜 | 6 格（2×3） |
| 15s | 7–9 镜 | 9 格（3×3） |
| 20s | 10–12 镜 | 12 格（3×4） |
| 30s | 15–20 镜 | 16–25 格 |

**详细规则：** `SKILL.md` →「无脚本模式」

---

## 如何选择？

| 你手头有什么 | 选哪种方式 |
|---|---|
| 已有脚本 | **方式 1** |
| 有参考视频 / 抖音链接 | **方式 2** |
| 只有产品图和卖点 | **方式 3** |

## 最终产出

无论哪种入口，完成后都会得到结构化 **资产包**，每个片段包含：

- `image_gen_input` — 宫格生图 JSON，可直接传给 `scripts/image/*`
- `video_gen_input` — 视频生成 JSON，填入宫格图路径后传给 `scripts/video/*`

需要真正出图出片时，按 `SKILL.md`「下游执行」章节操作即可。
